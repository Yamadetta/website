Реализация меню бургер без JavaScript

Реализация меню бургер с JavaScript происходит путём добавления класса на родительский элемент. К примеру, по клику мы добавляем класс .added-class , в следствие чего, отталкиваясь от данного класса нам необходимо стилизовать потомков данного класса. И таким образом, можно менять стилизацию целой группы элементов, просто добавив класс на родителя.

То есть, у нас есть родитель с классом .burger-menu, мы добавили этому элементу класс .added-class и стилизуем .burger-menu.added-class .navigation {}. Но того же самого поведения можно достичь, если вместо добавления нового класса воспользоваться псевдоклассом :checked для input. Так как input не имеет закрывающего тега, то остается резонный вопрос, каким образом можно стилизовать другие элементы с помощью псевдокласса для инпута?

И тут на помощь нам приходит комбинаторы ~ и +.

К примеру, сделаем input с id="burger-input". Ниже данного инпута разместим элементы, которые нам необходимо будет застилизовать. В данном случае я размещу label, который будет являться кнопкой-переключателем и саму навигацию.

<!-- Скрытый инпут вместо добавления класса -->
<input
  v-model="isShowNavigation"
  type="checkbox"
  id="burger-input"
/>

<!-- Кнопка, которая переключает инпут в true/false -->
<label class="burger" for="burger-input">
  <div class="burger__line"></div>
  <div class="burger__line"></div>
  <div class="burger__line"></div>
</label>

<!-- Сама навигация -->
<nav class="navigation" @click="hideNavigation">
  <ul>
    <li>
      <router-link class="link" to="/">Главная</router-link>
    </li>
    <li>
      <router-link class="link" to="/about">О проекте</router-link>
    </li>
    <li>
      <router-link class="link" to="/posts">Посты</router-link>
    </li>
    <li>
      <router-link class="link" to="/posts-store">Посты VueX</router-link>
    </li>
  </ul>
</nav

Поэтому вместо .burger-menu.added-class .navigation {}, мы можем использовать #burger-input:checked~.navigation {} и поведение будет точно таким же. А "переключать классы" будем через клик по label. Только классы в данном случае фактически не переключаются, т.к. мы используем псевдокласс.

Но в каком случае использовать +, а в каком ~?

Комбинатор + указывает на смежный или следующий селектор. То есть, фактически он выберет только один элемент, расположенный ниже него. Приведу пример. Есть у нас классический список и нам необходимо покрасить только второй элемент.

<ul>
  <li>Один</li>
  <li>Два</li>
  <li>Три</li>
</ul>

Указываем это в стилях:

li:first-of-type+li {
  color: red;
}

Получаем следующее:

  • Один
  • Два
  • Три

Комбинатор ~ наоборот, позволит захватить сразу все элементы ниже input, а не только один. Я третий раз обращаю внимание, что комбинаторы + и ~ работают только на элементах, которые располагаются ниже элемента, от которого мы отталкиваемся. В результате мы получим следующие стили. Привожу пример в scss синтаксисе, т.к. в нём легче всего стилизовать подобные элементы.

#burger-input {
  // input скрываем, т.к. он не нужен.
  display: none;
  &:checked {
    // Стилизуем навигацию с учётом псевдокласса. То есть, когда она будет вызвана пользователем.
    & ~ .navigation {
      transform: translateX(0);
    }

    & ~ .burger {
      .burger {
        &__line {
          transform-origin: left;

          &:nth-child(1) {
            width: 100%;
            transform: rotate(45deg);
          }

          &:nth-child(2) {
            width: 0;
          }
          &:nth-child(3) {
            transform: rotate(-45deg);
          }
        }
      }
    }
  }
}

Какие же плюсы данного подхода?

Нативная реализация на CSS не требует взаимодействия с DOM-деревом со стороны JS. Она особенно будет полезна в случае, если использование JS затруднено, либо невозможно. А также будет работать в браузерах, в которых отключен JavaScript.

Но у данного способа есть и соответствующие минусы.

В самом начале можно заметить, что хоть я и сказал, что при реализации не использовался JS. Но после мне пришлось создать модель для инпута и переключать состояние инпута с помощью Vue в двух случаях:

  1. клик по пункту меню (чтобы после клика меню автоматически закрывалось);
  2. клик по шапке.

И данную проблему можно было бы также решить с помощью CSS/HTML, обернув каждый пункт меню и логотип в шапке в label. Но это привело бы к усложнению вёрстки и снижению её читаемости. Поэтому я всё же решил сделать программизм.
